<p>The <b>Robot Exclusion Standard</b>, also known as the <b>Robots Exclusion Protocol</b> or <b>robots.txt protocol</b>, is a convention to prevent cooperating <a href="/wiki/Web_crawler" title="Web crawler">web crawlers</a> and other <a href="/wiki/Web_robot" title="Web robot" class="mw-redirect">web robots</a> from accessing all or part of a <a href="/wiki/Website" title="Website">website</a> which is otherwise publicly viewable. Robots are often used by <a href="/wiki/Search_engines" title="Search engines" class="mw-redirect">search engines</a> to categorize and archive web sites, or by webmasters to proofread source code. The standard is different from, but can be used in conjunction with, <a href="/wiki/Sitemaps" title="Sitemaps">Sitemaps</a>, a robot <i>inclusion</i> standard for websites.</p>

<h2><span class="editsection">[<a href="/w/index.php?title=Robots_exclusion_standard&amp;action=edit&amp;section=1" title="Edit section: History">edit</a>]</span> <span class="mw-headline" id="History">History</span></h2>
<p>The invention of "robots.txt" is attributed to Martijn Koster, when working for <a href="/wiki/WebCrawler" title="WebCrawler">WebCrawler</a> in 1994.<sup id="cite_ref-1" class="reference"><a href="#cite_note-1"><span>[</span>1<span>]</span></a></sup><sup id="cite_ref-2" class="reference"><a href="#cite_note-2"><span>[</span>2<span>]</span></a></sup> "robots.txt" was then popularized with the advent of <a href="/wiki/AltaVista" title="AltaVista">AltaVista</a>, and other popular search engines, in the following years.<sup class="Template-Fact" style="white-space:nowrap;">[<i><a href="/wiki/Wikipedia:Citation_needed" title="Wikipedia:Citation needed"><span title="This claim needs references to reliable sources from July 2011">citation needed</span></a></i>]</sup></p>
<h2><span class="editsection">[<a href="/w/index.php?title=Robots_exclusion_standard&amp;action=edit&amp;section=2" title="Edit section: About the standard">edit</a>]</span> <span class="mw-headline" id="About_the_standard">About the standard</span></h2>
<table class="metadata plainlinks ambox mbox-small-left ambox-content" style="">
<tr>
<td class="mbox-image"><a href="/wiki/File:Wiki_letter_w_cropped.svg" class="image"><img alt="[icon]" src="//upload.wikimedia.org/wikipedia/commons/thumb/1/1c/Wiki_letter_w_cropped.svg/20px-Wiki_letter_w_cropped.svg.png" width="20" height="14" srcset="//upload.wikimedia.org/wikipedia/commons/thumb/1/1c/Wiki_letter_w_cropped.svg/30px-Wiki_letter_w_cropped.svg.png 1.5x, //upload.wikimedia.org/wikipedia/commons/thumb/1/1c/Wiki_letter_w_cropped.svg/40px-Wiki_letter_w_cropped.svg.png 2x" /></a></td>
<td class="mbox-text" style=""><span class="mbox-text-span">This section requires <a class="external text" href="//en.wikipedia.org/w/index.php?title=Robots_exclusion_standard&amp;action=edit">expansion</a> with: which search engines follow that standard.. <small><i>(January 2011)</i></small></span></td>
</tr>
</table>
<p>If a site owner wishes to give instructions to web robots they must place a text file called <tt>robots.txt</tt> in the root of the web site hierarchy (e.g. <tt>www.example.com/robots.txt</tt>). This text file should contain the instructions in a specific format (see examples below). Robots that <i>choose</i> to follow the instructions try to fetch this file and read the instructions before fetching any other file from the web site. If this file doesn't exist web robots assume that the web owner wishes to provide no specific instructions.</p>
<p>A robots.txt file on a website will function as a request that specified robots ignore specified files or directories when crawling a site. This might be, for example, out of a preference for privacy from search engine results, or the belief that the content of the selected directories might be misleading or irrelevant to the categorization of the site as a whole, or out of a desire that an application only operate on certain data. Links to pages listed in robots.txt can still appear in search results if they are linked to from a page that is crawled.<sup id="cite_ref-3" class="reference"><a href="#cite_note-3"><span>[</span>3<span>]</span></a></sup></p>
<p>For websites with multiple subdomains, each subdomain must have its own robots.txt file. If <tt>example.com</tt> had a robots.txt file but <tt>a.example.com</tt> did not, the rules that would apply for <tt>example.com</tt> would not apply to <tt>a.example.com</tt>.</p>
<h2><span class="editsection">[<a href="/w/index.php?title=Robots_exclusion_standard&amp;action=edit&amp;section=3" title="Edit section: Disadvantages">edit</a>]</span> <span class="mw-headline" id="Disadvantages">Disadvantages</span></h2>
<p>Despite the use of the terms "allow" and "disallow", the protocol is purely advisory. It relies on the cooperation of the <a href="/wiki/Web_robot" title="Web robot" class="mw-redirect">web robot</a>, so that marking an area of a site out of bounds with robots.txt does not guarantee exclusion of all web robots. In particular, malicious web robots are unlikely to honor robots.txt.</p>
<p>While it is possible to prevent directory searches by anybody including web robots by setting up the security of the server properly, when the disallow directives are provided in the robots.txt file, the existence of these directories are disclosed to everyone.</p>
<p>There is no official standards body or <a href="/wiki/Request_for_Comments" title="Request for Comments">RFC</a> for the robots.txt protocol. It was <a rel="nofollow" class="external text" href="http://www.robotstxt.org/wc/norobots.html">created by consensus</a> in June 1994 by members of the robots mailing list (robots-request@nexor.co.uk). The information specifying the parts that should not be accessed is specified in a file called <b>robots.txt</b> in the top-level directory of the website. The robots.txt patterns are matched by simple substring comparisons, so care should be taken to make sure that patterns matching directories have the final '/' character appended, otherwise all files with names starting with that substring will match, rather than just those in the directory intended.</p>
<h2><span class="editsection">[<a href="/w/index.php?title=Robots_exclusion_standard&amp;action=edit&amp;section=4" title="Edit section: Alternatives">edit</a>]</span> <span class="mw-headline" id="Alternatives">Alternatives</span></h2>
<p>Many robots also pass a special <a href="/wiki/User-agent" title="User-agent" class="mw-redirect">user-agent</a> to the web server when fetching content<sup id="cite_ref-4" class="reference"><a href="#cite_note-4"><span>[</span>4<span>]</span></a></sup>. A web administrator could also configure the server to automatically return failure (or pass alternative content) when it detects a connection using one of the robots<sup id="cite_ref-5" class="reference"><a href="#cite_note-5"><span>[</span>5<span>]</span></a></sup><sup id="cite_ref-6" class="reference"><a href="#cite_note-6"><span>[</span>6<span>]</span></a></sup>.</p>
<h2><span class="editsection">[<a href="/w/index.php?title=Robots_exclusion_standard&amp;action=edit&amp;section=5" title="Edit section: Examples">edit</a>]</span> <span class="mw-headline" id="Examples">Examples</span></h2>
<p>This example tells <b>all robots</b> to visit <b>all files</b> because the wildcard <code>*</code> specifies all robots:</p>
<div dir="ltr" class="mw-geshi mw-code mw-content-ltr">
<div class="robots source-robots">
<pre class="de1">
<span class="kw1">User-agent</span><span class="sy0">:</span> *
<span class="kw1">Disallow</span><span class="sy0">:</span>
</pre></div>
</div>
<p>This example tells <b>all robots</b> to stay out of a website:</p>
<div dir="ltr" class="mw-geshi mw-code mw-content-ltr">
<div class="robots source-robots">
<pre class="de1">
<span class="kw1">User-agent</span><span class="sy0">:</span> *
<span class="kw1">Disallow</span><span class="sy0">:</span> /
</pre></div>
</div>
<p>The next is an example that tells <b>all robots</b> not to enter four directories of a website:</p>
<div dir="ltr" class="mw-geshi mw-code mw-content-ltr">
<div class="robots source-robots">
<pre class="de1">
<span class="kw1">User-agent</span><span class="sy0">:</span> *
<span class="kw1">Disallow</span><span class="sy0">:</span> /cgi-bin/
<span class="kw1">Disallow</span><span class="sy0">:</span> /images/
<span class="kw1">Disallow</span><span class="sy0">:</span> /tmp/
<span class="kw1">Disallow</span><span class="sy0">:</span> /private/
</pre></div>
</div>
<p>Example that tells <b>a specific robot</b> not to enter one specific directory:</p>
<div dir="ltr" class="mw-geshi mw-code mw-content-ltr">
<div class="robots source-robots">
<pre class="de1">
<span class="kw1">User-agent</span><span class="sy0">:</span> BadBot <span class="co1"># replace 'BadBot' with the actual user-agent of the bot</span>
<span class="kw1">Disallow</span><span class="sy0">:</span> /private/
</pre></div>
</div>
<p>Example that tells <b>all robots</b> not to enter one specific file:</p>
<div dir="ltr" class="mw-geshi mw-code mw-content-ltr">
<div class="robots source-robots">
<pre class="de1">
<span class="kw1">User-agent</span><span class="sy0">:</span> *
<span class="kw1">Disallow</span><span class="sy0">:</span> /directory/file.html
</pre></div>
</div>
<p>Note that all other files in the specified directory will be processed.</p>
<p>Example demonstrating how comments can be used:</p>
<div dir="ltr" class="mw-geshi mw-code mw-content-ltr">
<div class="robots source-robots">
<pre class="de1">
<span class="co1"># Comments appear after the "#" symbol at the start of a line, or after a directive</span>
<span class="kw1">User-agent</span><span class="sy0">:</span> * <span class="co1"># match all bots</span>
<span class="kw1">Disallow</span><span class="sy0">:</span> / <span class="co1"># keep them out</span>
</pre></div>
</div>
<p>It is also possible to list multiple <b>robot</b>s with their own rules. The actual <b>robot</b> string is defined by the crawler. A few sites, such as <a href="/wiki/Google" title="Google">Google</a>, support several user-agent strings that allow you to turn off a subset of their services by using specific user-agent strings.<sup id="cite_ref-7" class="reference"><a href="#cite_note-7"><span>[</span>7<span>]</span></a></sup></p>
<p>Example demonstrating multiple user-agents:</p>
<div dir="ltr" class="mw-geshi mw-code mw-content-ltr">
<div class="robots source-robots">
<pre class="de1">
<span class="kw1">User-agent</span><span class="sy0">:</span> googlebot        <span class="co1"># all services</span>
<span class="kw1">Disallow</span><span class="sy0">:</span> /private           <span class="co1"># disallow this directory</span>
 
<span class="kw1">User-agent</span><span class="sy0">:</span> googlebot-news   <span class="co1"># only the news service</span>
<span class="kw1">Disallow</span><span class="sy0">:</span> /                  <span class="co1"># on everything</span>
 
<span class="kw1">User-agent</span><span class="sy0">:</span> *                <span class="co1"># all robots</span>
<span class="kw1">Disallow</span><span class="sy0">:</span> /something         <span class="co1"># on this folder</span>
</pre></div>
</div>
<h2><span class="editsection">[<a href="/w/index.php?title=Robots_exclusion_standard&amp;action=edit&amp;section=6" title="Edit section: Nonstandard extensions">edit</a>]</span> <span class="mw-headline" id="Nonstandard_extensions">Nonstandard extensions</span></h2>
<h3><span class="editsection">[<a href="/w/index.php?title=Robots_exclusion_standard&amp;action=edit&amp;section=7" title="Edit section: Crawl-delay directive">edit</a>]</span> <span class="mw-headline" id="Crawl-delay_directive">Crawl-delay directive</span></h3>
<p>Several major crawlers support a <code>Crawl-delay</code> parameter, set to the number of seconds to wait between successive requests to the same server:<sup id="cite_ref-8" class="reference"><a href="#cite_note-8"><span>[</span>8<span>]</span></a></sup><sup id="cite_ref-9" class="reference"><a href="#cite_note-9"><span>[</span>9<span>]</span></a></sup><sup id="cite_ref-10" class="reference"><a href="#cite_note-10"><span>[</span>10<span>]</span></a></sup></p>
<div dir="ltr" class="mw-geshi mw-code mw-content-ltr">
<div class="robots source-robots">
<pre class="de1">
<span class="kw1">User-agent</span><span class="sy0">:</span> *
<span class="kw1">Crawl-delay</span><span class="sy0">:</span> <span class="nu0">10</span>
</pre></div>
</div>
<h3><span class="editsection">[<a href="/w/index.php?title=Robots_exclusion_standard&amp;action=edit&amp;section=8" title="Edit section: Allow directive">edit</a>]</span> <span class="mw-headline" id="Allow_directive">Allow directive</span></h3>
<p>Some major crawlers support an <code>Allow</code> directive which can counteract a following <code>Disallow</code> directive.<sup id="cite_ref-11" class="reference"><a href="#cite_note-11"><span>[</span>11<span>]</span></a></sup> <sup id="cite_ref-12" class="reference"><a href="#cite_note-12"><span>[</span>12<span>]</span></a></sup> This is useful when one tells robots to avoid an entire directory but still wants some HTML documents in that directory crawled and indexed. While by standard implementation the first matching robots.txt pattern always wins, Google's implementation differs in that Allow patterns with equal or more characters in the directive path win over a matching Disallow pattern.<sup id="cite_ref-13" class="reference"><a href="#cite_note-13"><span>[</span>13<span>]</span></a></sup> Bing uses the <code>Allow</code> or <code>Disallow</code> directive which is the most specific.<sup id="cite_ref-14" class="reference"><a href="#cite_note-14"><span>[</span>14<span>]</span></a></sup></p>
<p>In order to be compatible to all robots, if one wants to allow single files inside an otherwise disallowed directory, it is necessary to place the Allow directive(s) first, followed by the Disallow, for example:</p>
<div dir="ltr" class="mw-geshi mw-code mw-content-ltr">
<div class="robots source-robots">
<pre class="de1">
<span class="kw1">Allow</span><span class="sy0">:</span> /folder1/myfile.html
<span class="kw1">Disallow</span><span class="sy0">:</span> /folder1/
</pre></div>
</div>
<p>This example will Disallow anything in /folder1/ except /folder1/myfile.html, since the latter will match first. In case of Google, though, the order is not important.</p>
<h3><span class="editsection">[<a href="/w/index.php?title=Robots_exclusion_standard&amp;action=edit&amp;section=9" title="Edit section: Sitemap">edit</a>]</span> <span class="mw-headline" id="Sitemap">Sitemap</span></h3>
<p>Some crawlers support a <code>Sitemap</code> directive, allowing multiple <a href="/wiki/Sitemaps" title="Sitemaps">Sitemaps</a> in the same robots.txt in the form:<sup id="cite_ref-15" class="reference"><a href="#cite_note-15"><span>[</span>15<span>]</span></a></sup></p>
<div dir="ltr" class="mw-geshi mw-code mw-content-ltr">
<div class="robots source-robots">
<pre class="de1">
<span class="kw1">Sitemap</span><span class="sy0">:</span> http<span class="sy0">:</span>//www.gstatic.com/s2/sitemaps/profiles-sitemap.xml
<span class="kw1">Sitemap</span><span class="sy0">:</span> http<span class="sy0">:</span>//www.google.com/hostednews/sitemap_index.xml
</pre></div>
</div>
<h3><span class="editsection">[<a href="/w/index.php?title=Robots_exclusion_standard&amp;action=edit&amp;section=10" title="Edit section: Universal &quot;*&quot; match">edit</a>]</span> <span class="mw-headline" id="Universal_.22.2A.22_match">Universal "*" match</span></h3>
<p>The <i>Robot Exclusion Standard</i> does not mention anything about the "*" character in the <code>Disallow:</code> statement. Some crawlers like Googlebot and Slurp recognize strings containing "*", while MSNbot and Teoma interpret it in different ways.<sup id="cite_ref-16" class="reference"><a href="#cite_note-16"><span>[</span>16<span>]</span></a></sup></p>
